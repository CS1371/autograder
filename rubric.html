Rubric Specification

This is the specification for both the contents and the structure of the rubric archive to be used for grading.

If you aren't familiar with JSON, you'll need to learn that first...

  • What to Include
  • How to Organize it
  • JSON Specification

Quick Reference:

You'll need all of the following files:

  • Solution Files (.m format)
  • JSON file (rubric.json)
  • Variable files (.mat files)
  • Initializer files (.m files containing an initializer function definition)

What You'll Need

Solution Files

These files represent the homework solutions. They must be named without the _soln suffix, and must be the original .m file.

JSON Rubric

This file must be called rubric.json, and contains the JSON (specified later) for this homework set.

Variable Files

These files contain all the variables needed by a specific function (as specified in the rubric).

Initializer Files

If your code uses any initializers, be sure to include the function file in the solutions archive.

Supporting Files

If your code needs external file resources (i.e., a text file or image file) include these as well.

Quick Reference

  • Each problem should have its own folder
  • JSON rubric file should be at the top level

Detailed Structure

The organization of the archive is simple and intuitive.

At the top level of the archive, you should have:

  • A folder for the solutions (Solutions). This folder contains all the .m files.

  • A folder for the supporting files (SupportingFiles). This folder contains all .mat (variables), as well as any other supporting files.

    Note that the input variable mat file must end with either _rubrica.mat or _rubricb.mat.

  • The JSON files specifying the rubric, called rubric<a OR b>.json

A visual rendering of this format is shown below:

  • rubrica.json
  • Solutions
    • problemOne.m
      problemTwo.m
  • SupportingFiles
    • problemOneSupport____.m
      problemOneInputs.mat
      problemOneText.txt
      problemOneImg.png

The Rubric JSON

All fields must be present, though their value can be empty. Values can not be null
